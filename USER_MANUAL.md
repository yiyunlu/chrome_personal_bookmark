# TabHub 用户手册 / User Manual

TabHub 是一款 Chrome 新标签页书签管理扩展，让你在每次打开新标签页时都能快速访问和管理书签。

## 安装

1. 下载或构建扩展（参见 [README.md](./README.md)）
2. 打开 Chrome，访问 `chrome://extensions`
3. 右上角启用「开发者模式」
4. 点击「加载已解压的扩展程序」，选择 `dist/` 文件夹
5. 打开一个新标签页，TabHub 会自动替代默认的新标签页

> TabHub 使用 Chrome 原生书签 API 存储数据。如果你登录了 Chrome 帐号，书签会通过 Chrome 同步功能自动在设备间同步。

## 界面概览

TabHub 界面分为三个区域：

```
┌─────────────┬──────────────────────────────────────┐
│             │  工具栏（搜索、操作按钮）               │
│   侧 栏     ├──────────────────────────────────────┤
│             │                                      │
│  书签源      │  主内容区                              │
│  主题切换    │  （分类卡片 + 书签列表）                 │
│  分类列表    │                                      │
│             │                                      │
│             │                            💬 AI 助手 │
└─────────────┴──────────────────────────────────────┘
```

### 侧栏

- **书签源选择** — 切换不同的书签根文件夹（如书签栏、TabHub 等）
- **主题切换** — 跟随系统 / 浅色 / 深色三个选项
- **分类列表** — 显示当前书签源下的所有文件夹，点击可筛选
- **收起/展开** — 点击侧栏顶部的箭头按钮，收起为图标栏模式

### 工具栏

- **搜索框** — 输入关键词搜索书签标题或 URL（支持模糊搜索和语义匹配）
- **保存标签页** — 将当前窗口所有标签页保存为书签
- **自动整理** — 去除重复书签 + 按标题字母排序
- **AI 分类** — 使用 AI 分析书签并建议分类
- **失效检测** — 批量检测书签链接是否有效
- **管理模式** — 进入批量选择模式

### 主内容区

- 以卡片形式展示各分类及其书签
- 每个书签卡片显示标题、域名和网站图标
- 支持拖拽排序

## 基础操作

### 打开书签

直接点击书签卡片，会在当前标签页打开该链接。

### 搜索书签

1. 点击顶部搜索框（或按 `/` 键快速聚焦）
2. 输入关键词
3. 支持以下搜索方式：
   - **精确搜索** — 输入标题或 URL 的一部分，如 `react`
   - **模糊搜索** — 输入近似词，如 `reac` 也能匹配 `React Tutorial`
   - **分类搜索** — 输入类别名称，如 `social media` 会匹配 Twitter、Reddit 等社交网站
   - **域名搜索** — 输入域名，如 `github` 匹配所有 GitHub 书签

### 保存当前标签页

1. 点击工具栏的「保存标签页」按钮（或按 `S` 键）
2. 当前窗口所有打开的标签页会保存为书签
3. 自动创建一个以当前时间命名的新文件夹

### 切换书签源

在侧栏顶部的下拉菜单中选择不同的书签根文件夹。默认显示「书签栏」的内容。

## 书签管理

### 编辑书签

1. 右键点击书签卡片，选择「编辑」
2. 或将鼠标悬停在卡片上，点击铅笔图标
3. 在弹出的编辑窗口中修改：
   - **标题** — 书签显示名称
   - **URL** — 书签链接地址
   - **移动到文件夹** — 搜索并选择目标文件夹
4. 点击「保存」确认更改

### 删除书签

1. 右键点击书签卡片，选择「删除」
2. 或将鼠标悬停在卡片上，点击垃圾桶图标
3. 确认删除后，书签会移入 `.TabHub Trash` 回收文件夹
4. 底部会出现撤销提示（8 秒内可撤销）

### 拖拽排序

- **书签排序** — 将鼠标悬停在卡片左侧，出现拖拽手柄后拖动
- **分类排序** — 在侧栏中拖拽分类条目，或在主内容区拖拽分类卡片标题
- **跨分类移动** — 拖拽书签到另一个分类区域

> 注意：由于 Chrome 新标签页的兼容性限制，拖拽书签后页面会自动刷新。

### 管理文件夹

- **重命名** — 右键点击侧栏中的分类名，选择「重命名」
- **删除文件夹** — 右键点击侧栏中的分类名，选择「删除」（会删除文件夹及其所有书签）
- **折叠/展开** — 点击分类卡片标题旁的箭头

### 批量操作

1. 点击工具栏的「管理模式」按钮（或按 `M` 键）进入管理模式
2. 点击书签卡片选中（可多选）
3. 操作栏显示已选数量，可执行：
   - **批量移动** — 选择目标文件夹，一次移动多个书签
   - **批量删除** — 将选中的书签移入回收站
   - **清空选择** — 取消所有选中状态
4. 再次点击「退出管理」返回普通模式

### 自动整理

点击工具栏的「自动整理」按钮（或按 `O` 键）：

1. **去重** — 自动检测相同 URL 的重复书签，将副本移入回收站
2. **排序** — 在每个文件夹内按标题字母排序
3. 完成后显示整理结果，支持撤销

## AI 智能功能

### AI 智能分类

1. 点击工具栏的「AI 分类」按钮
2. 系统分析所有书签的 URL 和标题
3. 弹出建议窗口，显示每个建议的移动操作：
   - 书签名称
   - 建议的目标分类
   - 分类理由
4. 对每个建议可以：
   - 点击 ✓ **接受** — 标记为接受
   - 点击 ✕ **拒绝** — 忽略此建议
5. 确认后点击「应用 N 项」执行所有已接受的移动
6. 支持撤销

**分类规则（默认模式）：**

| 域名 | 分类 |
|------|------|
| github.com, stackoverflow.com, npmjs.com | Development |
| medium.com, dev.to, news.ycombinator.com | Reading |
| twitter.com, reddit.com, x.com | Social |
| youtube.com, spotify.com | Media |
| amazon.com | Shopping |
| figma.com, dribbble.com | Design |
| notion.so, docs.google.com | Productivity |

标题中包含 `react`、`vue`、`javascript`、`python` 等关键词的书签也会归入 Development。

### 失效链接检测

1. 点击工具栏的「失效检测」按钮
2. 系统逐批检测所有书签链接（每批 5 个，带进度条）
3. 检测完成后显示结果：
   - 如果全部有效，显示成功信息
   - 如果发现失效链接，列出详情
4. 对每个失效链接可以点击垃圾桶图标删除

### AI 助手聊天

点击右下角的聊天气泡按钮打开 AI 助手面板：

**支持的指令：**

| 指令 | 示例 | 说明 |
|------|------|------|
| 搜索 | `搜索 React` / `search tutorial` | 在书签中搜索关键词 |
| 移动 | `移动 GitHub 到 Development` | 将匹配的书签移动到指定分类 |
| 删除 | `删除 old bookmark` | 查找并删除匹配的书签 |
| 查找重复 | `查找重复` / `去重` / `duplicates` | 查找重复的书签 |
| 整理 | `整理` / `organize` | 提示使用工具栏整理功能 |
| 统计 | `统计` / `多少书签` / `count` | 显示书签和分类统计信息 |

- 对于移动和删除操作，AI 助手会先显示匹配结果，需点击「确认执行」才会执行
- 无法识别的输入会作为搜索处理

### 启用 Claude API（可选）

默认使用本地规则匹配。如需启用 Claude AI 增强功能：

1. 获取 Anthropic API Key（[console.anthropic.com](https://console.anthropic.com)）
2. 在 Chrome 开发者工具 Console 中执行：
   ```js
   chrome.storage.local.set({ tabhub_ai_api_key: '你的API密钥' })
   ```
3. 启用后，AI 分类、智能搜索和聊天助手将使用 Claude 进行更精准的语义理解

> 注意：API 调用会产生费用，请确保了解 Anthropic 的定价。API 密钥存储在本地浏览器中，不会发送到除 Anthropic API 以外的任何服务器。

## 主题设置

在侧栏中选择主题模式：

- **跟随系统** — 自动匹配操作系统的浅色/深色设置
- **浅色** — 白色背景
- **深色** — 深色背景

主题偏好会保存在浏览器中，下次打开新标签页时自动应用。

## 高级功能

### 书签源管理

TabHub 支持多个书签源（即 Chrome 书签栏下的顶级文件夹）。侧栏顶部的下拉菜单可在不同书签源之间切换。

- **TabHub** — 扩展自动创建的专用文件夹
- **书签栏** — Chrome 默认的书签栏
- 其他顶级文件夹也会显示为可选的书签源

### 回收站

删除的书签不会被永久删除，而是移入 `.TabHub Trash` 隐藏文件夹。如需恢复，可在 Chrome 书签管理器中找到该文件夹。

### 撤销操作

以下操作支持撤销（操作后 8 秒内）：

- 移动书签
- 编辑书签
- 删除书签
- 批量操作
- 自动整理
- AI 分类

底部会出现蓝色的撤销提示条，点击「撤销」即可恢复。

### 数据安全

- 所有数据存储在 Chrome 原生书签系统中
- 如果登录了 Chrome 帐号，数据通过 Chrome Sync 同步
- 扩展不会将书签数据发送到任何第三方服务器
- API 密钥（如已设置）仅存储在本地 `chrome.storage.local` 中

## 常见问题

### Q: 书签数据存储在哪里？

书签直接使用 Chrome 原生书签系统存储。你可以在 Chrome 书签管理器（`chrome://bookmarks`）中看到 TabHub 创建的文件夹结构。

### Q: 卸载扩展后书签会丢失吗？

不会。书签是 Chrome 原生数据，卸载扩展不影响书签。

### Q: 拖拽书签后页面会刷新，这是正常的吗？

是的。这是为了确保在 Chrome 新标签页环境中拖拽操作的稳定性而采取的兼容性措施。

### Q: AI 分类不准确怎么办？

默认模式使用简单的域名和关键词规则匹配，可能不完全准确。你可以：
- 手动接受或拒绝每个建议
- 设置 Claude API 密钥以获得更智能的分类

### Q: 如何恢复误删的书签？

在操作后 8 秒内点击底部的「撤销」按钮。超时后，可在 Chrome 书签管理器中找到 `.TabHub Trash` 文件夹手动恢复。

### Q: 搜索支持哪些语言？

搜索支持任何语言的标题和 URL 匹配。分类关键词扩展（如 "social media"→Twitter）目前以英文为主。

### Q: 如何完全重置 TabHub？

在 Chrome 书签管理器中删除 `TabHub` 文件夹即可。下次打开新标签页时会自动重新创建。
